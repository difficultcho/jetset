# Handoff: AURELLE 时尚小程序（高保真设计交接）

## 概述

AURELLE 是一个奢侈时尚品牌的移动端小程序设计，包含 **35+ 个页面/状态** 的完整购物体验：
五个主 Tab（首页、关于品牌、商城、我的、线下门店）、商品列表/详情、购物袋、下单支付全流程、会员注册开卡、订单管理、心愿单/足迹、门店查找、帮助中心等。

设计语言：极简奢侈品美学 —— 大面积留白、细字重无衬线字体、超宽字距的品牌字标、黑白灰 + 暖沙色调、细分割线（0.5px）、无圆角或极小圆角。

## 关于设计文件

本交接包中的 `AURELLE 时尚小程序.dc.html` + `support.js` 是 **HTML 设计参考稿**（可直接双击在浏览器中打开交互体验），**不是可直接上线的生产代码**。

你的任务是：**在目标代码库的现有技术环境中重新实现这套设计**。
- 若目标是微信小程序：用原生 WXML/WXSS 或 Taro/uni-app 实现
- 若目标是 Web/H5：用现有框架（React/Vue 等）及其组件规范实现
- 若尚无代码库：推荐 React + 移动端 H5（或 Taro 以便多端发布）

HTML 文件中所有页面写在一个文件内，通过状态切换显示；实现时应拆分为独立路由页面。

## 保真度

**高保真（hifi）**。颜色、字号、字距、间距、布局、交互均为最终意图，应按像素级精度还原。
唯一例外：所有**图片均为占位块**（CSS 斜纹渐变 + 等宽字体说明标签），需替换为真实摄影图（见「资源」一节）。

- 状态栏：32px（时间 + 信号/电量，深色内容）

导航为**栈式**：`nav(screen)` 入栈、返回箭头出栈；切换 Tab 清空栈。

```

 ├─ 走马灯(hero) ──────────→ campaign（广告大片）
 ├─ 探索HIGH SUMMER系列 ──→ seriesList（系列商品列表）★
 ├─ 探索广告大片(×2) ─────→ campaign
 ├─ 商品走马灯(单品) ─────→ pdp（商品详情）
 ├─ 探索鞋履 ────────────→ list(cat=shoes)
 └─ 探索包袋 ────────────→ list(cat=bags)
Tab: 关于品牌 brand
 ├─ 视频块#1(林间白裙) ──→ 原地播放/暂停（不跳转）★
 │   └─ 下方标题文字 ────→ campaign
 ├─ A.PROJECTS ─────────→ projects
 ├─ A.MOMENTS ──────────→ moments
 ├─ 品牌故事 ────────────→ story
 └─ 走进精品店 ──────────→ storeIntro → storeDetail
Tab: 商城 shop（左侧11个类目切换右侧内容）★
 ├─ 大图卡片 → seriesList / list / campaign（按类目）
 └─ 瓷片 → list(对应品类)；无货品类 → Toast“即将上架”
Tab: 我的 me（未注册态/已注册态）
 ├─ 注册\登录 → joinOpen底部弹层 → openCard开卡表单 → 注册成功
 ├─ 头像✎ → profile；二维码icon → memberCode
 ├─ 我的订单 orders / 我的预约 bookings / 心愿单 wishlist / 我的足迹 footprints
 └─ 门店导购 guide / 地址管理 addresses / 附近门店 nearby /
    个人设置 settings / 帮助中心 help→helpDetail→sizeGuide / 关注我们 follow
Tab: 线下门店 stores
 ├─ FIND A STORE → nearby（搜索+省市筛选）→ storeDetail
```

## 交互与动效汇总

| 交互 | 规格 |
|---|---|
| 走马灯 | 自动轮播 4s/帧（可配 2–8s），`transform: translateX` 过渡 0.6s ease；圆点指示器同步；实现时请补充手势滑动支持 |
| 筛选抽屉 | 右滑入 `translateX(100%)→0` 0.28s ease + 黑色 35% 遮罩 fadeIn 0.25s |
| 注册弹层 | 底部上滑 `translateY(100%)→0` 0.3s ease + 40% 遮罩 |
| 品牌视频块 | 点击图面切换播放/暂停；播放态=双竖条按钮 + 底部 3px 进度条 30s 线性充满 |
| Toast | fadeIn 0.2s，停留 1.8s |
| 支付倒计时 | 订单创建时刻 +30 分钟 deadline，每秒刷新 `00:mm:ss`，支付页与订单详情页共用 |
| 加购 | 同 pid+size 合并 qty+1；角标 = Σqty；跨页面实时同步 |
| 收藏 | 星标即时切换 ★/☆ + Toast「已加入/移出心愿单」 |
| 足迹 | 每次打开 pdp 将商品 id 插入足迹首位（去重，上限 20）|
| 提交拦截 | 未勾协议、未选地址、未勾选购物袋商品 → 对应 Toast，不跳转 |

## 状态管理与数据模型

持久化（localStorage key `aurelle_mp_v1`，实现时换成后端/小程序 Storage）：

```ts
{
  registered: boolean,
  user: { nick, name, phone, gender, birthday, region, email, code },
  addresses: { name, phone, full, def: boolean }[],
  cartItems: { pid, size, qty, checked: boolean }[],
  orders: { id, time, status: 'pending'|'cancelled', pid, size, qty,
            total, addr, deadline: epochMs }[],
  wishlist: string[],   // 商品id
  footprints: string[], // 商品id，最新在前
  agreedDate: string
}
```

商品模型（示例数据 13 件，完整见 HTML 内 `DB.products`）：

```ts
{ id, name, sub, price, sale?, cat: 'dresses'|'tops'|'shoes'|'bags'|'jewellery',
  color, colorHex, desc, bullets: string[], code /* 款号 */ }
```

会话内状态：当前 Tab、导航栈、走马灯索引、商城选中类目、列表排序（综合/新品）、筛选抽屉开合、PDP 选中尺码、手风琴开合、品牌视频播放态、支付倒计时。
